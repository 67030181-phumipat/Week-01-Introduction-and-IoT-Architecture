# สัปดาห์ที่ 1: แนะนำรายวิชาและสถาปัตยกรรมของ IoT

### **ทฤษฎี**
ภาพรวมระบบ IoT, บทบาทของเซ็นเซอร์ และบอร์ดไมโครคอนโทรลเลอร์ (เช่น ESP32 / Raspberry Pi Pico)

### 1. ภาพรวมระบบ IoT (IoT Ecosystem Overview)

อินเทอร์เน็ตของสรรพสิ่ง (Internet of Things) ไม่ใช่แค่การนำอุปกรณ์มาต่อ Wi-Fi แต่คือระบบนิเวศที่บูรณาการฮาร์ดแวร์ ซอฟต์แวร์ และเครือข่ายเข้าด้วยกัน เพื่อให้อุปกรณ์สามารถสลับและแลกเปลี่ยนข้อมูลเพื่อตัดสินใจได้โดยอัตโนมัติ สถาปัตยกรรมหลักแบ่งออกเป็น 4 ชั้น (4-Tier Architecture):
- **Perception Layer (ชั้นรับรู้/ฮาร์ดแวร์):** เปรียบเสมือนประสาทสัมผัส ทำหน้าที่เก็บข้อมูลจากโลกกายภาพ (เซ็นเซอร์) และสั่งการอุปกรณ์ (แอคชูเอเตอร์)
- **Network Layer (ชั้นเครือข่าย):** ทำหน้าที่รับส่งข้อมูลผ่านโปรโตคอลและตัวกลางต่าง ๆ เช่น Wi-Fi, Bluetooth, LoRaWAN, หรือ Cellular
- **Middleware/Cloud Processing Layer (ชั้นประมวลผลข้อมูล):** ระบบบริหารจัดการข้อมูล ฐานข้อมูล และการวิเคราะห์ผลบนระบบคลาวด์
- **Application Layer (ชั้นประยุกต์ใช้งาน):** หน้าต่างเชื่อมต่อกับผู้ใช้ (User Interface) เช่น Web Dashboard, โมบายแอปพลิเคชัน หรือระบบแจ้งเตือน
### 2. บทบาทของเซ็นเซอร์และทรานสดิวเซอร์ (The Role of Sensors & Transducers)
- **Transducer (ทรานสดิวเซอร์):** ตัวแปลงพลังงานจากรูปหนึ่งไปสู่อีกรูปหนึ่ง (เช่น แปลงความร้อน แสง หรือแรงกด ไปเป็นพลังงานไฟฟ้า)
- **Sensor (เซ็นเซอร์):** อุปกรณ์ที่ใช้ตรวจจับการเปลี่ยนแปลงของสภาพแวดล้อมทางกายภาพ แล้วส่งสัญญาณเอาต์พุตออกมาเป็นค่าที่ระบบประมวลผลเข้าใจได้
- **ความสำคัญในระบบ IoT:** เซ็นเซอร์คือ "ต้นน้ำ" ของข้อมูล หากปราศจากเซ็นเซอร์ที่แม่นยำ ระบบคลาวด์หรือ AI ปลายน้ำก็จะไม่สามารถนำข้อมูลไปประมวลผลเพื่อสร้างประโยชน์ได้เลย

### 3. บอร์ดไมโครคอนโทรลเลอร์สำหรับงาน IoT (Edge Devices)
ในการเรียนรู้ยุคปัจจุบัน มีบอร์ดควบคุมหลักที่เป็นที่นิยมสูง 2 ตระกูล ซึ่งมีข้อดีและสถาปัตยกรรมที่แตกต่างกัน:
- **ESP32 Series (Espressif Systems):** * เป็นชิปราคาประหยัดที่ทรงพลัง มีโมดูล Wi-Fi และ Bluetooth (BLE) มาให้ในตัว
    - เหมาะสำหรับงาน IoT ส่วนใหญ่เพราะสามารถเชื่อมต่ออินเทอร์เน็ตได้ทันทีโดยไม่ต้องต่ออุปกรณ์เสริม
- **Raspberry Pi Pico W (Raspberry Pi Foundation):**
    - ใช้ชิป RP2040 ที่พัฒนาขึ้นเอง เด่นเรื่องการจัดการพลังงาน ขาไอโอ (GPIO) ที่ยืดหยุ่น และมีรุ่น "W" ที่รองรับ Wi-Fi
    - เหมาะสำหรับการเรียนรู้ทั้งภาษา C++ และ MicroPython

### **ปฏิบัติ** 
การติดตั้งเครื่องมือ (VS Code) และทบทวนการเขียนโปรแกรมควบคุมพื้นฐาน (Hello World / Blink)
### 1. การเตรียมพร้อมและติดตั้งเครื่องมือ (Environment Setup)
ผู้เรียนจะได้เลือกและติดตั้งเครื่องมือตามความถนัด หรือตามแนวทางของห้องปฏิบัติการ:
- **Option A: Arduino IDE** เครื่องมือมาตรฐานระดับเริ่มต้น ใช้งานง่าย เรียนรู้ไวผ่านภาษา C/C++ มีระบบ Library Manager ที่ช่วยให้ดึงโค้ดของเซ็นเซอร์ต่าง ๆ มาใช้งานได้สะดวก
- **Option B: VS Code + PlatformIO** เครื่องมือระดับมืออาชีพ (Advanced) เหมาะสำหรับโครงงานขนาดใหญ่ มีระบบ Code Completion, Debugging และการจัดการโครงสร้างไฟล์ที่เป็นระบบมากกว่า
> 🛠️ **กิจกรรมการทดลองที่ 1.1:** การติดตั้งโปรแกรม, การดาวน์โหลดและติดตั้งบอร์ดไดรเวอร์ (ESP32 / Pico Board Manager) และการลงไดรเวอร์สำหรับชิปแปลงสัญญาณพอร์ต USB (เช่น CH340 / CP210x) เพื่อให้คอมพิวเตอร์มองเห็นบอร์ดไมโครคอนโทรลเลอร์
### 2. การเขียนโปรแกรมควบคุมพื้นฐาน (First Code Execution)
การเรียนรู้โครงสร้างพื้นฐานของสถาปัตยกรรมซอฟต์แวร์บนไมโครคอนโทรลเลอร์ผ่าน 2 ตัวอย่างคลาสสิก:
- **โจทย์ที่ 1: "Hello World" ผ่าน Serial Monitor**
    - **วัตถุประสงค์:** เรียนรู้การสื่อสารแบบอนุกรม (Serial Communication) ระหว่างบอร์ดกับคอมพิวเตอร์เพื่อใช้ในการตรวจซ่อมและดูค่าข้อมูล (Debugging)
    - **แนวคิดโค้ด:** กำหนดความเร็วบัส (Baud rate เช่น 115200) ในฟังก์ชัน `setup()` และใช้คำสั่ง `Serial.println()` ในการส่งข้อความออกหน้าจอ
- **โจทย์ที่ 2: "Blink" (ไฟกระพริบ)**
    - **วัตถุประสงค์:** เรียนรู้โครงสร้างคำสั่งควบคุมการทำงานของขาสัญญาณดิจิทัล (Digital Output) และจังหวะเวลา (Timing/Delay)
    - **แนวคิดโค้ด:** ฝึกใช้คำสั่งกำหนดสถานะขา `pinMode(LED_BUILTIN, OUTPUT)`, การสั่งเปิด/ปิดไฟ `digitalWrite()` สูง (HIGH) และต่ำ (LOW) สลับกันโดยหน่วงเวลาด้วย `delay(1000)`
