**Internet Layer (OSI Layer 3)** คือชั้นที่ทำหน้าที่จัดการการส่งข้อมูลจากต้นทางไปยังปลายทางผ่านเครือข่าย โดยใช้การกำหนดที่อยู่ (addressing) และการกำหนดเส้นทาง (routing)

ในระบบ IoT ชั้นนี้มีความสำคัญมาก เพราะอุปกรณ์ IoT มีจำนวนมหาศาลและต้องการระบบที่สามารถจัดการ address และ routing ได้อย่างมีประสิทธิภาพ

เทคโนโลยีสำคัญใน Internet Layer สำหรับ IoT ได้แก่:
- **IPv6**
- **6LoWPAN**
- **RPL**

เราจะอธิบายทีละตัว

#  IPv6 ในระบบ IoT

## 1) Addressing ใน Internet Layer

ภายใต้ **Internet Layer (OSI Layer 3)** อุปกรณ์ทุกตัวในเครือข่ายจะถูกกำหนดหมายเลขประจำตัวด้วย **IP address** เพื่อให้สามารถ:
- ระบุตัวตนของอุปกรณ์ (identify)
- ส่งข้อมูลไปยังปลายทางที่ถูกต้อง (route)
- เชื่อมต่อกับอุปกรณ์อื่นในเครือข่าย

ในระบบ IoT ซึ่งมีอุปกรณ์จำนวนมหาศาล การเลือกใช้รูปแบบ address จึงเป็นเรื่องสำคัญมาก

#  ทำไม IoT จึงนิยมใช้ IPv6 มากกว่า IPv4
## 2) ข้อจำกัดของ IPv4

IPv4 ใช้ address ขนาด **32-bit** จำนวน address ทั้งหมดประมาณ:
$$2^{32}≈4.3 พันล้านอุปกรณ์$$

จำนวนนี้เพียงพอสำหรับยุคคอมพิวเตอร์และสมาร์ตโฟน แต่ **ไม่เพียงพอสำหรับยุค IoT** ที่มีอุปกรณ์จำนวนมหาศาล เช่น:
- เซนเซอร์ในเมืองอัจฉริยะ
- เซนเซอร์ในเกษตรกรรม
- อุปกรณ์ในบ้านอัจฉริยะ
- อุปกรณ์ในโรงงาน
- อุปกรณ์ติดตามทรัพย์สิน

IoT ต้องการ address มากกว่าหลายร้อยเท่า

## 3) ความสามารถของ IPv6

IPv6 ใช้ address ขนาด **128-bit** จำนวน address ทั้งหมดประมาณ:

$$2^{128} ≈ 340\quad ล้านล้านล้านล้านอุปกรณ์$$
ซึ่งมากพอสำหรับ:
- อุปกรณ์ IoT ทุกชิ้นบนโลก
- อุปกรณ์ในอนาคตอีกหลายร้อยปี
- การกำหนด address แบบไม่ซ้ำแม้ในระบบขนาดใหญ่ระดับประเทศ

นอกจากนี้ IPv6 ยังมีความสามารถที่เหมาะกับ IoT เช่น:
- **Stateless Address Autoconfiguration (SLAAC)** → อุปกรณ์กำหนด address เองได้
- **โครงสร้าง address ที่ยืดหยุ่น** → เหมาะกับเครือข่ายขนาดใหญ่
- **รองรับการทำงานร่วมกับ 6LoWPAN** → ทำให้ IPv6 ใช้บนเครือข่ายพลังงานต่ำได้
- **รองรับความปลอดภัยระดับ IPsec โดยกำเนิด**

# แต่ IoT ส่วนใหญ่ “ไม่ใช้ public IPv6 address”

แม้ IPv6 จะรองรับอุปกรณ์จำนวนมหาศาล แต่ในระบบ IoT จริง:

### **อุปกรณ์ IoT ส่วนใหญ่ไม่ต้องการ public address**

เหตุผลคือ:
- อุปกรณ์ IoT มีพลังงานต่ำ
- ไม่ต้องการให้โลกภายนอกเข้าถึงโดยตรง
- ต้องการความปลอดภัยสูง
- การจัดการ public address มีความซับซ้อน
- อุปกรณ์จำนวนมากอยู่ในเครือข่ายท้องถิ่น (local network)

### **อุปกรณ์ IoT มักเข้าถึงโลกภายนอกผ่าน Gateway**

Gateway ทำหน้าที่:
- แปลงโปรโตคอล
- รวมข้อมูลจากอุปกรณ์หลายตัว
- จัดการ routing
- จัดการความปลอดภัย
- เชื่อมต่อกับ cloud ผ่าน IPv6 หรือ IPv4

ดังนั้นแม้ IoT จะใช้ IPv6 เป็นหลักในระดับ network แต่ **การสื่อสารกับอินเทอร์เน็ตมักผ่าน gateway ไม่ใช่อุปกรณ์โดยตรง**

#  สรุป IPv6 สำหรับ IoT

| ประเด็น        | รายละเอียด                                |
| -------------- | ----------------------------------------- |
| Address        | IPv6 ใช้ 128-bit รองรับอุปกรณ์จำนวนมหาศาล |
| IPv4           | จำกัดที่ 32-bit (4.3 พันล้าน address)     |
| IoT            | นิยมใช้ IPv6 เพราะรองรับ massive IoT      |
| Public address | IoT ส่วนใหญ่ไม่ใช้ public IPv6            |
| Gateway        | เป็นตัวกลางเชื่อม IoT กับโลกภายนอก        |

#  6LoWPAN (IPv6 Low Power Wireless Personal Area Network)

## 1) ความหมายของ 6LoWPAN

ชื่อเต็มของ 6LoWPAN คือ **IPv6 Low Power Wireless Personal Area Network**

หมายถึง **มาตรฐานที่อนุญาตให้ใช้ IPv6 บนเครือข่ายไร้สายพลังงานต่ำ** เช่น IEEE 802.15.4 ซึ่งเป็นเครือข่ายที่มี:
- MTU เล็กมาก (127 bytes)
- พลังงานต่ำ
- อุปกรณ์ขนาดเล็ก
- Bandwidth ต่ำ
- ระยะสื่อสารใกล้–ปานกลาง

โดยปกติ IPv6 packet มีขนาดใหญ่เกินไปสำหรับเครือข่ายแบบนี้ แต่ 6LoWPAN ทำให้ IPv6 สามารถทำงานได้อย่างมีประสิทธิภาพ

#  6LoWPAN ทำงานบน IEEE 802.15.4

6LoWPAN เป็นมาตรฐานที่ “ทำให้ IPv6 ทำงานบน 802.15.4 wireless networks ได้”
802.15.4 คือพื้นฐานของหลายเทคโนโลยี IoT เช่น:
- ZigBee
- Thread
- Wireless sensor networks

แต่เดิมเทคโนโลยีเหล่านี้ไม่สามารถใช้ IPv6 ได้โดยตรง เพราะ:
- IPv6 header ใหญ่มาก
- MTU ของ 802.15.4 เล็กมาก
- อุปกรณ์มีพลังงานจำกัด

6LoWPAN จึงเข้ามาแก้ปัญหาเหล่านี้

#  6LoWPAN ทำอะไรบ้าง?

### **1) Header Compression**

บีบอัด IPv6 header ให้เล็กลงอย่างมาก ทำให้สามารถส่ง IPv6 packet ผ่านเครือข่ายพลังงานต่ำได้

### **2) Fragmentation**

แบ่ง IPv6 packet ออกเป็นหลายส่วน เพื่อให้เหมาะกับ MTU ขนาดเล็กของ 802.15.4

### **3) Adaptation Layer**

เพิ่มชั้นพิเศษระหว่าง Layer 2 และ Layer 3 เพื่อให้ IPv6 ทำงานบนเครือข่ายพลังงานต่ำได้อย่างราบรื่น

### **4) รองรับ Routing แบบ IoT (เช่น RPL)**

ช่วยให้เครือข่ายเซนเซอร์สามารถใช้ routing แบบ IPv6 ได้

#  การใช้งาน 6LoWPAN ในโลก IoT

6LoWPAN นิยมใช้กับอุปกรณ์ที่ต้องการพลังงานต่ำและสื่อสารข้อมูลขนาดเล็ก เช่น:

### **Wireless Sensor Networks (WSN)**

เช่น
- เซนเซอร์วัดอุณหภูมิ
- เซนเซอร์วัดความชื้น
- เซนเซอร์ตรวจจับการเคลื่อนไหว
- เซนเซอร์ในระบบเกษตรอัจฉริยะ

### **Home Automation Devices**

เช่น
- อุปกรณ์สมาร์ตโฮม
- เซนเซอร์ประตู/หน้าต่าง
- ระบบควบคุมไฟ
- ระบบตรวจจับควัน
- ระบบควบคุมพลังงานภายในบ้าน

6LoWPAN ทำให้ระบบเหล่านี้สามารถใช้ IPv6 ได้โดยตรง ซึ่งช่วยให้:
- การเชื่อมต่อกับอินเทอร์เน็ตง่ายขึ้น
- การจัดการ address เป็นมาตรฐาน
- การทำงานร่วมกับระบบอื่นในอนาคตง่ายขึ้น
- รองรับอุปกรณ์จำนวนมากในเครือข่ายเดียว

#  สรุป 6LoWPAN

6LoWPAN คือเทคโนโลยีที่ทำให้ IPv6 สามารถทำงานบนเครือข่ายไร้สายพลังงานต่ำอย่าง IEEE 802.15.4 ได้ โดยใช้การบีบอัด header และการแบ่ง packet เพื่อให้เหมาะกับอุปกรณ์ IoT ขนาดเล็ก

| คุณสมบัติ | รายละเอียด                                          |
| --------- | --------------------------------------------------- |
| ชื่อเต็ม  | IPv6 Low Power Wireless Personal Area Network       |
| ทำงานบน   | IEEE 802.15.4                                       |
| หน้าที่   | ทำให้ IPv6 ใช้บนเครือข่ายพลังงานต่ำได้              |
| ใช้ใน     | Wireless sensor networks, Home automation           |
| จุดเด่น   | Header compression, Fragmentation, Adaptation layer |

#  RPL (ออกเสียงว่า “ripple”)

## 1) ทำไมต้องมี RPL

ในระบบ IoT ที่ใช้ IPv6 บนอุปกรณ์พลังงานต่ำ เช่นเครือข่าย:
- Wireless sensor networks
- Home automation networks
- Smart building
- Smart agriculture
- Smart city

อุปกรณ์มักมีข้อจำกัดสำคัญ:
- พลังงานต่ำมาก
- สัญญาณไม่เสถียร (lossy)
- topology เปลี่ยนบ่อย
- packet loss สูงและคาดเดาไม่ได้
- อุปกรณ์บางตัวหลุดจากเครือข่ายเป็นช่วง ๆ
- bandwidth ต่ำ

เครือข่ายลักษณะนี้เรียกว่า **Low-Power and Lossy Networks (LLNs)** ซึ่ง **routing แบบทั่วไป (เช่น OSPF, RIP)** ไม่สามารถใช้งานได้

ดังนั้นจึงต้องมี routing protocol ที่ออกแบบมาเฉพาะสำหรับ IoT → **RPL**

#  RPL คืออะไร

RPL (Routing Protocol for Low-Power and Lossy Networks) เป็นโปรโตคอล routing ที่ออกแบบโดย IETF เพื่อใช้กับ IPv6 ในเครือข่ายพลังงานต่ำที่มีความสูญเสียสูง

คุณสมบัติหลัก:
- ทำงานบน IPv6
- รองรับเครือข่ายพลังงานต่ำ
- รองรับ packet loss สูง
- topology เปลี่ยนได้แบบ dynamic
- ใช้พลังงานน้อยที่สุด
- ลดเวลาในการสื่อสาร (latency)
- รองรับอุปกรณ์จำนวนมาก

RPL จึงเป็น routing protocol มาตรฐานสำหรับ IoT networks ที่ใช้ 6LoWPAN, ZigBee IP, Thread และเครือข่ายเซนเซอร์อื่น ๆ

#  ปัญหาของเครือข่าย IoT ที่ RPL เข้ามาแก้ไข

### **1) Lossy Networks**

อุปกรณ์พลังงานต่ำมักส่งสัญญาณเบา ทำให้เกิด packet loss สูงและไม่สามารถคาดเดาได้

### **2) การเชื่อมต่อไม่ต่อเนื่อง**

เซนเซอร์บางตัวอาจหลับ (sleep mode) เพื่อประหยัดพลังงาน ทำให้ topology เปลี่ยนตลอดเวลา

### **3) พลังงานจำกัด**

อุปกรณ์ต้องเลือกเส้นทางที่ใช้พลังงานน้อยที่สุด

### **4) latency ต้องต่ำ**

บางงาน เช่น home automation ต้องตอบสนองเร็ว

RPL ถูกออกแบบมาเพื่อแก้ปัญหาเหล่านี้โดยตรง

# RPL ทำงานอย่างไร

RPL สร้างโครงสร้าง routing แบบ dynamic โดยใช้กราฟชนิดหนึ่งเรียกว่า:

## **DODAG (Destination-Oriented Directed Acyclic Graph)**

ลักษณะสำคัญ:
- เป็นกราฟแบบไม่มี loop
- มีจุดหมายปลายทาง (root) เช่น gateway
- node ทุกตัวจะเลือกเส้นทางที่ดีที่สุดไปยัง root
- เส้นทางถูกคำนวณใหม่เมื่อ topology เปลี่ยน
- ใช้ metric หลายแบบ เช่น
    - พลังงานที่เหลือ
    - ความเสถียรของลิงก์
    - latency
    - จำนวน hop

### **RPL เลือกเส้นทางโดยคำนึงถึงสองสิ่งสำคัญ:**
- **minimize energy** → ใช้พลังงานน้อยที่สุด
- **minimize latency time** → เสียเวลาน้อยที่สุด

จึงเหมาะกับงานที่ต้องการความเสถียรและประหยัดพลังงานพร้อมกัน

#  การใช้งาน RPL ในโลกจริง

RPL นิยมใช้ในเครือข่าย:
### **Wireless Sensor Networks**
เช่น
- เซนเซอร์วัดอุณหภูมิ
- เซนเซอร์วัดความชื้น
- เซนเซอร์ตรวจจับการเคลื่อนไหว
- เซนเซอร์ในระบบเกษตรอัจฉริยะ

### **Home Automation**

เช่น
- ระบบไฟอัจฉริยะ
- เซนเซอร์ประตู/หน้าต่าง
- ระบบตรวจจับควัน
- ระบบควบคุมพลังงานในบ้าน

### **Smart Building / Smart City**

เช่น
- ระบบตรวจวัดคุณภาพอากาศ
- ระบบตรวจวัดพลังงาน
- ระบบตรวจวัดน้ำท่วม
- ระบบควบคุมไฟถนน

RPL คือ routing protocol ที่ทำให้เครือข่าย IoT ขนาดใหญ่สามารถทำงานได้อย่างมีประสิทธิภาพแม้จะมีข้อจำกัดสูง

# สรุป RPL

RPL เป็น routing protocol ที่ออกแบบมาเฉพาะสำหรับเครือข่าย IoT พลังงานต่ำและมีความสูญเสียสูง โดยใช้โครงสร้างแบบ DODAG เพื่อคำนวณเส้นทางที่ดีที่สุดแบบ dynamic โดยคำนึงถึงพลังงานและ latency

| คุณสมบัติ | รายละเอียด                                         |
| --------- | -------------------------------------------------- |
| ออกเสียง  | ripple                                             |
| ทำงานบน   | IPv6                                               |
| ใช้กับ    | Low-Power and Lossy Networks                       |
| จุดเด่น   | Dynamic routing, minimize energy, minimize latency |
| ใช้ใน     | WSN, Home automation, Smart building, Smart city   |
